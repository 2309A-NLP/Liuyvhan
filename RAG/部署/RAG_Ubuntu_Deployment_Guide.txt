RAG 项目 Ubuntu 部署文档

版本说明：按当前项目真实联调结果整理
文档时间：2026-05-29
项目仓库：https://github.com/Lyh1221/RAG.git
默认分支：main

一、文档适用范围

本部署文档基于当前项目实际代码、实际依赖和已跑通方案整理，不是通用模板。

当前项目已验证通过的真实链路包括：
1. MySQL
2. Redis
3. Milvus
4. 本地 embedding 服务
5. 本地 rerank 服务
6. 主应用 FastAPI
7. 阿里云百炼大模型接口
8. `/chat`
9. `/chat/stream`

当前项目主服务端口如下：
1. 主应用：8001
2. Embedding 服务：8002
3. Rerank 服务：8003
4. Milvus：19530
5. Redis：6379
6. MySQL：3306

二、当前项目部署特点

1. 主应用通过 OpenAI-compatible 接口访问大模型。
2. 当前真实跑通的是阿里云百炼：
   - `LLM_PROVIDER=aliyun`
   - `LLM_MODEL_NAME=qwen2.5-7b-instruct`
   - `LLM_API_BASE=https://dashscope.aliyuncs.com/compatible-mode/v1`
3. 当前本地 embedding 与 rerank 是单独 FastAPI 服务。
4. embedding 维度为 768。
5. 知识库依赖 Milvus，短期记忆依赖 Redis，角色与用户信息依赖 MySQL。
6. 本文默认部署目标为 Ubuntu 22.04 / 24.04。

三、建议服务器规格

1. Ubuntu 22.04 LTS 或 24.04 LTS
2. 内存建议 16GB 以上，推荐 32GB
3. 系统盘及数据盘建议预留 100GB 以上
4. 若后续要本地部署 vLLM / SGLang，可额外准备 NVIDIA GPU

说明：
当前这套项目部署并不强制要求 GPU，因为你现在真实跑通的是阿里云百炼接口，embedding/rerank 也可在 CPU 上运行。

五、Ubuntu 安装 Docker

```bash
sudo apt update
sudo apt install -y ca-certificates curl gnupg lsb-release
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu \
  $(. /etc/os-release && echo $VERSION_CODENAME) stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo systemctl enable docker
sudo systemctl start docker
sudo usermod -aG docker $USER
newgrp docker

docker --version
docker compose version
```

六、Python 环境

方案 A：uv

1. 安装 uv：

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
source ~/.local/bin/env
uv --version
```

2. 创建项目环境：

```bash
cd /opt/rag/runtime/RAG
uv venv --python 3.12 --seed
source .venv/bin/activate
```

3. 安装依赖：

```bash
uv pip install -r requirements.txt
```

方案 B：conda

```bash
conda create -n rag python=3.12 -y
conda activate rag
pip install -r requirements.txt
```

进入项目目录：

```bash

```

建议使用下面这份部署模板：

```bash
cat > /opt/rag/runtime/RAG/.env <<'EOF'
APP_PORT=8001
APP_HOST=0.0.0.0

DB_BACKEND=mysql
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=your_mysql_password
MYSQL_DATABASE=ragdata
MYSQL_CHARSET=utf8mb4

REDIS_ENABLED=true
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_DB=0

MILVUS_ENABLED=true
MILVUS_URI=http://127.0.0.1:19530
MILVUS_TOKEN=

LLM_PROVIDER=aliyun
LLM_MODEL_NAME=qwen2.5-7b-instruct
LLM_API_BASE=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_API_KEY=your_aliyun_dashscope_api_key
LLM_TIMEOUT=120
LLM_MAX_TOKENS=1024
LLM_TEMPERATURE=0.35
LLM_TOP_P=0.9
LLM_PRESENCE_PENALTY=0.2

REDIS_ENABLED=true
MILVUS_ENABLED=true

EMBEDDING_BACKEND=remote_api
EMBEDDING_MODEL_NAME=moka-ai/m3e-base
EMBEDDING_API_URL=http://127.0.0.1:8002/embeddings
EMBEDDING_API_KEY=
EMBEDDING_API_TIMEOUT=30
EMBEDDING_DIMENSION=768
LOCAL_EMBEDDING_MODEL_PATH=/opt/rag/models/m3e-base
LOCAL_EMBEDDING_SERVICE_HOST=0.0.0.0
LOCAL_EMBEDDING_SERVICE_PORT=8002

RERANK_BACKEND=remote_api
RERANK_MODEL_NAME=BAAI/bge-reranker-v2-m3
RERANK_API_URL=http://127.0.0.1:8003/rerank
RERANK_API_KEY=
RERANK_API_TIMEOUT=30
LOCAL_RERANK_MODEL_PATH=BAAI/bge-reranker-v2-m3
LOCAL_RERANK_SERVICE_HOST=0.0.0.0
LOCAL_RERANK_SERVICE_PORT=8003

PDF_AUTO_IMPORT_ENABLED=true
PDF_AUTO_IMPORT_DEFAULT_ROLE_ID=doctor
PDF_PARSER_BACKEND=mineru
MINERU_API_URL=https://mineru.net/api/v4/file-urls/batch
MINERU_API_TOKEN=your_mineru_api_token
MINERU_BACKEND=hybrid-auto-engine
MINERU_PARSE_METHOD=auto
MINERU_LANGUAGE=ch
MINERU_FORMULA_ENABLE=true
MINERU_TABLE_ENABLE=true
MINERU_IMAGE_ANALYSIS=true
MINERU_TIMEOUT=300
EOF
```

说明：
1. 不要把真实 API Key 写进部署文档或提交到仓库。
2. 如果不使用 MinerU，可将 `PDF_PARSER_BACKEND` 改为 `pymupdf`，并清空 `MINERU_API_URL` 与 `MINERU_API_TOKEN`。

十、部署 Redis / MySQL / Milvus

1. Redis

```bash
docker rm -f rag-redis 2>/dev/null || true
docker run -d \
  --name rag-redis \
  --restart unless-stopped \
  -p 6379:6379 \
  -v /opt/rag/redis:/data \
  redis:7-alpine
```

2. MySQL

```bash
docker rm -f rag-mysql 2>/dev/null || true
docker run -d \
  --name rag-mysql \
  --restart unless-stopped \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=your_mysql_password \
  -e MYSQL_DATABASE=ragdata \
  -v /opt/rag/mysql:/var/lib/mysql \
  mysql:8.4
```

3. Milvus

```bash
cd /opt/rag/runtime/RAG
docker compose -f docker-compose.milvus.yml up -d
```

检查状态：

```bash
docker ps
```

5. 当前本地 embedding 与 rerank 是真实可用的

十七、常见故障排查

1. `/chat` 返回 401
优先检查：
- `LLM_API_KEY` 是否正确
- `LLM_API_BASE` 是否为阿里云百炼兼容地址
- 模型名是否为百炼支持的模型名

2. `/chat` 能返回但 `references` 为空
优先检查：
- `scripts/load_demo_data.py` 是否已执行
- `role_knowledge` 中该角色是否已有知识数据
- `role_id` 是否填写正确

3. embedding 启动失败
优先检查：
- `LOCAL_EMBEDDING_MODEL_PATH` 是否正确
- 模型目录是否存在

4. rerank 启动失败
优先检查：
- `LOCAL_RERANK_MODEL_PATH` 是否可访问
- 当前机器内存是否足够

5. Milvus 无法命中
优先检查：
- `MILVUS_URI`
- collection 是否初始化
- embedding 维度与现有 collection 是否一致

十八、建议的生产化补充
